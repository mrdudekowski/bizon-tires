## 🎯 Назначение каждой папки

### `assets/`
Все статические ресурсы сайта. Организованы по типам для легкого поиска и управления.

### `css/`
Модульная структура стилей:
- **components/** - каждый компонент в отдельном файле
- **utilities/** - переиспользуемые утилиты
- **responsive/** - медиа-запросы для адаптивности
- **variables.css** - единый источник для всех CSS переменных

### `js/`
Модульная структура JavaScript:
- **modules/** - функциональные модули (навигация, формы, анимации)
- **utils/** - вспомогательные функции
- **config.js** - единый источник конфигурации

### `data/`
JSON файлы с данными:
- Легко обновлять без изменения кода
- Можно загружать динамически
- Готовность к интеграции с API

### `sections/`
Зарезервировано для будущей модульности HTML.

### `docs/`
Вся документация проекта в одном месте.

## 📝 Примечания

- Старые файлы (`style.css`, `script.js`) можно оставить для обратной совместимости
- Новая структура готова к использованию
- Все файлы следуют принципу Single Source of Truth
- Структура масштабируема и легко расширяема
