# Brainstorm UI/UX для лендинга БИЗОН ТИРС

## Секции, для которых прорабатываются идеи
- Header / Навигация
- Hero
- О нас (`#about`)
- Преимущества (`#advantages`)
- Гарантии / Соц.доказательства (`#trust`)
- Форма заявки (`#order`)
- Footer
- Sticky CTA / глобальные конверсионные элементы

---

## Header / Навигация — 5 идей
1. **Строгая top-bar с ключевыми метриками**  
   Узкая полоса над основным хедером с лаконичными метриками: "10+ лет на рынке", "5000+ клиентов", "Доставка по РФ". Легкое появление сверху при загрузке (fade + slide-down, 200–250 мс) без излишней динамики.

2. **Навигация с подсветкой контекста секции**  
   При скролле к каждой секции соответствующий пункт меню подсвечивается тонкой нижней линией и изменением насыщенности шрифта, без резкой смены цвета. Анимация линии — краткий underline-slide слева направо.

3. **Минималистичный sticky-header после первого скролла**  
   Полноразмерный хедер при старте, а после прокрутки на 1 экран он трансформируется в компактный sticky-хедер: уменьшается логотип, скрывается часть текста, остаются только ключевые пункты и CTA-кнопка. Плавная трансформация (height, font-size) за ~200 мс.

4. **Ховер-состояния с микроанимацией иконок**  
   При наведении на пункты меню маленькие иконки (грузовик, щит, диаграмма) появляются слева от текста с лёгким fade-in и смещением по оси X на 2–4 пикселя. Это подчеркивает смысл раздела, не нарушая строгий стиль.

5. **Контрастный, но сдержанный контактный блок в хедере**  
   Блок телефона и режима работы: телефон — крупнее, режим работы — меньшим шрифтом под ним. При ховере на телефон — лёгкая пульсация обводки иконки телефона (outline-grow), чтобы аккуратно подсказать, что сюда можно кликнуть.

## Hero — 5 идей
1. **Глубокий фон с параллаксом на основе индустриального пейзажа**  
   Статичный основной фон + лёгкий параллакс для крупных объектов (силуэт грузовика, склад, дорога). Смещение в пределах 5–10px при движении мыши или скролле, без навязчивости.

2. **Линейный прогресс-бар "пути клиента" под заголовком**  
   Под основным заголовком разместить горизонтальную схему: "Заявка → Подбор → Поставка". При загрузке hero каждый шаг подсвечивается по очереди (progress-indicator), занимая в сумме не более 1–1.5 секунд.

3. **Анимация появления заголовка в формате "строчного" набора**  
   Заголовок проявляется не буквами, а строками: сначала основная строка, затем подзаголовок с лёгким задержанным fade-up (50–80 мс между элементами). Без карикатурных typing-эффектов.

4. **Главная CTA-кнопка с серьёзным "индустриальным" ховером**  
   При наведении на основную кнопку: лёгкий градиент-сдвиг, псевдо-"подсветка" по нижнему краю и небольшая тень, как у тяжёлой металлической кнопки. Анимация — 150–200 мс, без увеличения шрифта.

5. **Информационный блок "Для кого" в виде строгих бейджей**  
   Под CTA разместить 3–4 бейджа: "Грузовые автопарки", "Логистические компании", "Строительные компании". При ховере — смена фона на чуть более светлый и плавное появление небольшой подписи (tooltip-like) над бейджем.

## О нас (`#about`) — 5 идей
1. **Карточки преимуществ с вертикальной линией таймлайна**  
   Иконки и заголовки выровнены по общей вертикальной линии, соединённой тонким акцентным штрихом. При скролле к секции элементы появляются снизу вверх вдоль этой линии (staggered fade-up).

2. **Смена иконок на более "инженерные" пиктограммы**  
   Вместо эмодзи — строгие монохромные иконки (контуры шин, щит, грузовик). При ховере толщине линий добавляется +1px и появляется тонкая внутренняя тень.

3. **Чёткая визуальная иерархия: "мы" vs "вы"**  
   В каждом блоке добавить микроабзац "Что это даёт вам" — акцентным цветом и меньшим кеглем. Никакой анимационной перегрузки, только лёгкий fade-in этого текста с задержкой.

4. **Интерактивное раскрытие деталей в один клик**  
   Под каждым пунктом — ссылку "Подробнее" с аккуратным раскрытием дополнительного текста (accordion-inline). Анимация высоты блока — 150–200 мс, без резких скачков.

5. **Маркировка ключевых фактов бейджами**  
   Внутри текста выделять числа и факты малым контрастным бейджем (например, "500+ моделей", "10+ лет"). Без анимации при загрузке, только лёгкий hover-shadow для бейджа.

## Преимущества (`#advantages`) — 5 идей
1. **Нумерованные блоки как строгие "этапы испытаний"**  
   Оформить номера 01–04 как крупные, полупрозрачные цифры на фоне каждой карточки, создавая ощущение технического отчёта. При ховере цифра становится чуть более контрастной.

2. **Лёгкая анимация прогресс-баров внутри карточек**  
   Для некоторых преимуществ (износостойкость, экономия топлива) добавить горизонтальные прогресс-индикаторы (например, 90%, 80%). Они заполняются один раз при появлении секции в зоне видимости.

3. **Сравнительные подписи "Обычная шина vs БИЗОН ТИРС"**  
   Внутри описаний 2–3 строки-компаратора с иконками стрелок: красная узкая линия для "обычно", зелёная широкая — для "мы". Анимация — мягкое проявление линий слева направо.

4. **Группировка карточек в логические блоки**  
   Визуально разделить преимущества на группы: "Надёжность", "Экономика", "Безопасность" с маленькими caption-заголовками над группами. Никакой сложной анимации, только последовательное появление групп.

5. **Встроенные подсказки по кейсам применения**  
   Иконка вопроса рядом с заголовком преимущества: при наведении — маленький popover с примером реального сценария ("Фуры на дальних рейсах", "Самосвалы на стройке"). Popover появляется с небольшой задержкой и мягким fade.

## Гарантии / Соц.доказательства (`#trust`) — 5 идей
1. **Анимация счётчиков с реальными порогами**  
   Статистика (10+, 5000+, 500+) заполняется счётчиками один раз при первом показе секции. Анимация не бешеная: 0.7–1 сек, шаги сглажены (ease-out).

2. **Сетка гарантий как "карты политики качества"**  
   Три блока гарантий оформить как карточки с небольшой иконкой "shield" и монохромным фоном. При ховере — аккуратное свечение границы (inner-shadow), создающее ощущение защищённости.

3. **Тонкая линия времени под статистикой**  
   Под блоком статов — линия с чекпоинтами по годам (например, 2014, 2018, 2024). При скролле линия заполняется слева направо, а чекпоинты последовательно подсвечиваются.

4. **Встраиваемые логотипы клиентов/партнёров (строгий монохром)**  
   Логотипы размещаются в один ряд или карусель, но в монохромном исполнении. При ховере — лёгкий возврат к фирменным цветам конкретного логотипа.

5. **Краткие цитаты-кейсы вместо длинных отзывов**  
   2–3 лаконичных кейс-цитаты: кто клиент, какой парк, какой результат (например, "–5% затрат на шины"). Появляются по очереди с лёгким fade и смещением по оси Y при скролле.

## Форма заявки (`#order`) — 5 идей
1. **Прогресс-индикатор "1 шаг до подбора"**  
   Над формой — минималистичный прогресс-бар с текстом "1 шаг до подбора оптимальной резины". При фокусе на первом поле бар заполняется на 33%, на втором — 66%, на третьем — 100%.

2. **Интеллектуальные подсказки по полям**  
   Подписи к полям с маленькими серыми подсказками: "Нужны реальные контактные данные — так менеджер быстрее свяжется". Подсказки появляются только при фокусе и исчезают при вводе.

3. **Подтверждение отправки в стиле "инженерного отчёта"**  
   После успешной отправки — строгий блок с чеклистом: "Заявка принята", "Данные проверены", "Ответ в течение 15 минут" с галочками и плавной анимацией их появления.

4. **Упор на конфиденциальность рядом с чекбоксом**  
   Рядом с чекбоксом политики — маленькая иконка-замок; при наведении всплывает краткое резюме: "Не передаём третьим лицам. Используем только для связи".

5. **Микроанимация кнопки отправки по состояниям**  
   Состояния: обычное, hover, "отправка" (spinner справа от текста), "успех" (галочка) — всё в одном размере кнопки, без скачков ширины. Переходы между состояниями плавные (150–200 мс).

## Footer — 5 идей
1. **Чёткое разделение зон: Контакты / Информация / Режим**  
   Использовать визуальные разделители (тонкие вертикальные линии) и выровнять все заголовки. Никаких лишних декоративных элементов, только аккуратная сетка.

2. **Мини-карта или схематичное указание географии**  
   Маленькая схематичная карта РФ с лёгким подсвечиванием регионов, где уже есть клиенты (heatmap в 2–3 уровня). Без интерактива, только визуальный акцент.

3. **Блок "Юридическая чистота"**  
   Добавить сдержанный текстовый блок с ИНН/ОГРН и упоминанием официального статуса поставщика. Можно снабдить маленькой иконкой документа.

4. **Системный паттерн ховеров для ссылок**  
   Все ссылки в футере при наведении получают одинаковый underline-animation (слева направо) и лёгкое изменение насыщенности цвета.

5. **Финальная микро-CTA внизу**  
   В строке копирайта добавить лаконичную ссылку "Партнёрские программы для оптовых клиентов" — второстепенный, но заметный путь конверсии для B2B.

---

## Sticky CTA / глобальные конверсионные элементы — 5 идей
1. **Контекстный текст в sticky-CTA в зависимости от секции**  
   Текст на кнопке меняется в зависимости от текущего блока: в секции преимуществ — "Рассчитать экономию", в доверии — "Получить консультацию по парку". Логика реализуется через JS, а анимация смены текста — мягкий crossfade.

2. **Незначимое, но ощутимое "подтягивание" кнопки при скролле**  
   При активном скролле sticky-кнопка чуть смещается вверх/вниз с небольшим лагом (spring-like easing), создавая ощущение физического объекта, но не отвлекая от контента.

3. **Встроенный индикатор "Ответим за 15 минут"**  
   Внутри sticky-CTA добавить маленькую строку под основным текстом с иконкой часов. Это усиливает ценностное предложение без расширения отдельного блока.

4. **Сдержанное появление/исчезновение sticky-CTA**  
   Кнопка не видна в hero (чтобы не конкурировать с основным CTA) и появляется только после прокрутки вниз. Плавный fade + slide-up на 150–200 мс.

5. **Системные подсказки по конверсии в ключевых местах**  
   У ссылок, ведущих к форме, при первом наведении появляется небольший tooltip "Подбор шин под ваш парк — бесплатно". Работает только один раз за сессию, чтобы не раздражать.
