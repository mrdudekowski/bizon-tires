# Архитектура лендинга БИЗОН ТИРС

## Обзор

Лендинг построен на принципах модульности, масштабируемости и поддержки Single Source of Truth.

## Структура файлов

### Корневой уровень
- `index.html` - главная страница, точка входа
- `README.md` - основная документация проекта

### Assets (Ресурсы)
Все статические файлы организованы по типам:

#### `assets/images/`
- `hero/` - изображения для главного экрана
- `products/` - фотографии продукции
- `advantages/` - иллюстрации преимуществ
- `testimonials/` - фото для отзывов
- `logos/` - логотипы партнеров и брендов
- `icons/` - иконки и пиктограммы

#### `assets/fonts/`
Шрифты проекта (если используются кастомные)

#### `assets/videos/`
Видео материалы (презентации, обзоры продукции)

#### `assets/documents/`
PDF файлы: каталоги, сертификаты, инструкции

### CSS (Стили)

#### Модульная структура
- `css/main.css` - главный файл, импортирует все остальные
- `css/variables.css` - CSS переменные (цвета, отступы, размеры)
- `css/components/` - стили отдельных компонентов
- `css/utilities/` - утилитарные классы и сброс стилей
- `css/responsive/` - медиа-запросы для адаптивности

#### Принцип организации CSS
Каждый компонент имеет свой файл, что упрощает поддержку и поиск нужных стилей.

### JavaScript

#### Модульная структура
- `js/main.js` - точка входа, инициализация
- `js/config.js` - конфигурация (API endpoints, настройки)
- `js/modules/` - функциональные модули
- `js/utils/` - вспомогательные функции

#### Модули
- `form.js` - обработка и валидация форм
- `navigation.js` - навигация и скролл
- `animations.js` - анимации и эффекты
- `validation.js` - валидация данных

### Sections (Секции)
HTML-шаблоны секций для возможной модульности в будущем.

### Data (Данные)
JSON файлы с данными:
- `products.json` - информация о продукции
- `advantages.json` - преимущества
- `config.json` - общая конфигурация

## Принципы работы

### Single Source of Truth
- Все данные о продукции в `data/products.json`
- Конфигурация в `data/config.json`
- CSS переменные в `css/variables.css`

### Модульность
- Каждый компонент изолирован
- Легко добавлять новые секции
- Простое тестирование отдельных частей

### Масштабируемость
- Структура готова к добавлению новых функций
- Легко интегрировать сборщик (Webpack, Vite)
- Готовность к переходу на фреймворк при необходимости

## Порядок загрузки

1. HTML загружается
2. CSS переменные и базовые стили
3. Компоненты подключаются по мере необходимости
4. JavaScript инициализируется после загрузки DOM
