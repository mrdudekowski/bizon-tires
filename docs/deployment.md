# Инструкции по деплою

## Подготовка к деплою

### 1. Проверка файлов
Убедитесь, что все файлы на месте:
- `index.html` - главная страница
- `css/main.css` - стили
- `js/main.js` - JavaScript
- Все ресурсы в папке `assets/`

### 2. Оптимизация

#### Изображения
- Сожмите все изображения (используйте TinyPNG, ImageOptim и т.д.)
- Используйте современные форматы (WebP, AVIF)
- Добавьте lazy loading для изображений

#### CSS и JS
- Минифицируйте файлы (если не используется сборщик)
- Удалите неиспользуемый код

### 3. Тестирование
- Проверьте на разных устройствах
- Протестируйте формы
- Проверьте производительность (PageSpeed Insights)

## Варианты деплоя

### GitHub Pages
1. Создайте репозиторий на GitHub
2. Загрузите файлы
3. Включите GitHub Pages в настройках репозитория
4. Сайт будет доступен по адресу: `https://username.github.io/repository-name`

### Netlify
1. Зарегистрируйтесь на netlify.com
2. Перетащите папку проекта в Netlify
3. Сайт будет автоматически задеплоен

### Vercel
1. Установите Vercel CLI: `npm i -g vercel`
2. В папке проекта выполните: `vercel`
3. Следуйте инструкциям

### Обычный веб-сервер
1. Загрузите все файлы на сервер через FTP/SFTP
2. Убедитесь, что `index.html` находится в корневой директории
3. Проверьте права доступа к файлам (обычно 644 для файлов, 755 для папок)

## После деплоя

1. Проверьте работу сайта
2. Настройте домен (если нужно)
3. Настройте SSL сертификат (HTTPS)
4. Настройте аналитику (Google Analytics, Яндекс.Метрика)
5. Проверьте индексацию в поисковых системах
